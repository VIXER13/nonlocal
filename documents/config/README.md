# Структура конфигурационных файлов

Для запуска программы NonLocFEM необходимо указать путь к исполняемому файлу и в качестве аргумента передать путь к JSON файлу с конфигурацией запуска, например
```bash
/path/to/NonLocFEM /path/to/config.json
```

Конфигурационный файл `config.json` должен содержать в себе набор определённых ключевых полей, каждое из которых обладает собственной структурой, которая может разниться в зависимости от постановки задачи или параметров. Ниже представлены определения основных полей и их структуры.

## Основные поля
```json
{
    "task": {...},
    "mesh": {...},
    "save": {...},
    "auxiliary": {...},
    "time": {...},
    "boundaries": {...},
    "materials": {...}
}
```
- [`task`](#task-field) (обязательное) --- основная постановка задачи;
- [`mesh`](#mesh-field) (условно обязательное) --- данные о сетке: пути, конфигурации конечных элементов и т.д. (обязательное, когда размерность задачи больше 1);
- [`save`](#save-field) (обязательное) --- параметры сохранения результатов и метаинформации;
- [`auxiliary`](#auxiliary-field) (опциональное) --- вспомогательные параметры: правая часть, начальные распределения, доп. условия в некоторых постановках и т.д.;
- [`time`](#time-field) (условно обязательное) --- параметры времени в нестационарных расчётах (обязательное, когда [`time_dependency`](#task-field) равно `true`);
- [`boundaries`](#boundaries-field) (обязательное) --- граничные условия;
- [`materials`](#materials-field) (обязательное) --- параметры материалов.


<a id="task-field"></a>

### Поле `task`
```json
{
    "dimension": N,
    "problem": "type",
    "time_dependency": false
}
```

<a id="task-field"></a>

- `dimension` (обязательное) --- размерность задачи, может принимать значения `1` или `2`;
- `problem` (обязательное) --- тип задачи, может принимать значения `"thermal"`, `"mechanical"` и `"thermomechanical"`;
- `time_dependency` (обязательное) --- зависимость от времени, может принимать значения `true` или `false`.


<a id="mesh-field"></a>

### Поле `mesh`
Для случая, когда [`dimension`](#task-field) равен `1` имеет структуру
```json
{
    "element_order": "order",
    "quadrature_order": "order"
}
```
- `element_order` (опциональное) --- порядок элемента, может принимать значения: `"linear"`, `"quadratic"`, `"cubic"`, `"quartic"`, `"quintic"` или числовое значение от `1` до `5` включительно. Значение по умолчанию: `"linear"` (или `1` в числовом представлении).
- `quadrature_order` (опциональное) --- порядок квадратуры, может принимать значения: `"linear"`, `"quadratic"`, `"cubic"`, `"quartic"`, `"quintic"` или числовое значение от `1` до `5` включительно. Значение по умолчанию: совпадает с `element_order`.

Для случая, когда [`dimension`](#task-field) равен `2` имеет структуру
```json
{
    "path": "/path/to/mesh.su2"
}
```
- `path` (обязательное) --- путь к файлу с конечно-элементной сеткой в формате su2.


<a id="save-field"></a>

### Поле `save`
```json
{
    "folder": "/path/to/folder",
    "config": "name",
    "csv": "name",
    "vtk": "name",
    "precision": N
}
```
- `folder` (опциональное) --- путь к папке, в которую сохранять результаты. Значение по умолчанию: путь к текущей рабочей директории;
- `config` (опциональное) --- имя под которым сохранить текущий конфигурационный файл в папке `folder`. Значение по умолчанию: в случае пропуска не сохраняет файл;
- `csv` (опциональное) --- имя под которым сохранить результаты расчётов в формате CSV в папке `folder`. Значение по умолчанию: в случае пропуска не сохраняет файл;
- `vtk` (опциональное) --- имя под которым сохранить результаты расчётов в формате VTK в папке `folder` (для случая, когда [`dimension`](#task-field) равен `2`). Значение по умолчанию: в случае пропуска не сохраняет файл;
- `precision` (опциональное) --- точность чисел с плавающей точкой при сохранении результатов. Значение по умолчанию: использует точность по умолчанию.

<a id="time-field"></a>

### Поле `time`
```json
{
    "time_step": tau,
    "initial_time": t,
    "steps_count": N,
    "save_frequency": K
}
```
- `time_step` (обязательное) --- размер шага по времени;
- `initial_time` (опциональное) --- начальный момент времени. Значение по умолчанию: 0;
- `steps_count` (обязательное) --- количество шагов по времени;
- `save_frequency` (опциональное) --- частота сохранения результатов. Значение по умолчанию: 1.


<a id="boundaries-field"></a>

### Поле `boundaries`
Для случая, когда [`dimension`](#task-field) равен `1` имеет структуру
```json
{
    "left": {...},
    "right": {...}
}
```
- `left` (обязательное) --- граничное условие на левой границе;
- `right` (обязательное) --- граничное условие на правой границе.

Для случая, когда [`dimension`](#task-field) равен `2` имеет структуру
```json
{
    "condition_name_1": {...},
    ...
    "condition_name_N": {...}
}
```
- `conditionN` (опциональные) --- граничные условия. Названия полей совпадают с названиями, определёнными в файле с [конечно-элементной сеткой](#mesh-field);

Структура граничных условий, помимо размерности, ещё зависит от типа решаемой задачи.

#### Тепловая задача
```json
{
    "condition_name": {
        "kind": "type",
        "temperature": T,
        "flux": F,
        "heat_transfer": alpha,
        "emissivity": eps
    }
}
```
- `kind` (обязательное) --- определяет род граничного условия, может принимать значения: `"temperature"`, `"flux"`, `"convection"`, `"radiation"`, `"combined"`;
- `temperature` (условно обязательное) --- значение температуры или температуры окружающей среды на границе (обязательное, когда `kind` равно `temperature` или `convection`);
- `flux` (условно обязательное) --- значение теплового потока на границе (обязательное, когда `kind` равно `flux`);
- `heat_transfer` (условно обязательное) --- значение коэффициента конвективного теплообмена (обязательное, когда `kind` равно `convection`);
- `emissivity` (условно обязательное) --- значение коэффициента излучения (обязательное, когда `kind` равно `radiation`)

<details>
<summary>Примеры</summary>

- Температурное граничное условие:
    ```json
    {
        "kind": "temperature",
        "temperature": 487.0
    }
    ```
- Потоковое условие:
    ```json
    {
        "kind": "flux",
        "flux": -100.0
    }
    ```
- Конвективный теплообмен:
    ```json
    {
        "kind": "convection",
        "temperature": 250.0,
        "heat_transfer": 5000.0
    }
    ```
- Температурное излучение:
    ```json
    {
        "kind": "radiation",
        "emissivity": 0.7
    }
    ```
- Комбинированные граничные условия (линейная комбинация теплового потока, конвективного теплообмена и излучения):
    ```json
    {
        "kind": "combined",
        "flux": -100.0,
        "temperature": 250.0,
        "heat_transfer": 5000.0,
        "emissivity": 0.7
    }
    ```
</details>


<a id="materials-field"></a>

### Поле `materials`
Для случая, когда [`dimension`](#task-field) равен `1` имеет структуру
```json
[
    {
        "elements_count": N1,
        "length": L1,
        "physical": {...},
        "model": {...}
    },
    ...
    {
        "elements_count": Nn,
        "length": Ln,
        "physical": {...},
        "model": {...}
    },
]
```

Для случая, когда [`dimension`](#task-field) равен `2` имеет структуру
```json
{
    "material1": {
        "physical": {...},
        "model": {...}
    },
    ...
    "materialN": {
        "physical": {...},
        "model": {...}
    }
}
```
